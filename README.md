- **L'equipe humaine** : pour comprendre l'idee, savoir quoi preparer, quoi dire, quoi eviter, et comment defendre le projet.

## 4. Le probleme principal

Les entreprises modernes ont acces a beaucoup d'outils : WhatsApp, Google Sheets, CRM, email, sites web, publicites, plateformes e-commerce, calendriers, bases de donnees, etc.

Mais dans la pratique, beaucoup d'entreprises restent bloquees dans des workflows manuels.

Exemples concrets :

- Un vendeur Instagram recoit des messages toute la journee sur WhatsApp.
- Il doit repondre aux memes questions.
- Il doit confirmer les commandes Cash on Delivery.
- Il doit verifier le stock.
- Il doit recopier le nom, le telephone, l'adresse et le produit dans Google Sheets.
- Il doit relancer les clients.
- Il doit suivre les commandes.
- Il doit faire tout cela vite, sans erreurs, meme tard le soir.

Ce n'est pas un probleme de "manque de logiciel".

Le vrai probleme est que les outils ne travaillent pas seuls.

L'humain reste le lien entre tous les outils.

L'humain lit le message.

L'humain comprend.

L'humain copie.

L'humain colle.

L'humain confirme.

L'humain met a jour.

L'humain relance.

L'humain est devenu le goulot d'etranglement.

Le modele IA analyse la demande.

## 12. Architecture technique plus professionnelle

Un assistant IA ou un membre technique peut presenter l'architecture de cette facon :

1. **Canal d'entree**
   WhatsApp, Email, Slack, Discord, site web, formulaire, API.

2. **Connecteur**
   Module qui recoit les messages et les transforme en evenement exploitable.

3. **Runtime agentique**
   Environnement dans lequel l'agent fonctionne, avec ses regles, son role, ses limites et ses outils.

4. **Modele IA**
   Frontier model comme Claude, GPT ou autre modele de qualite.

5. **Orchestrateur**
   Systeme qui decide si une tache doit etre executee directement ou deleguee a plusieurs sous-agents.

6. **Couche de securite**
   RBAC, validation d'intention, permissions minimales, logs, filtres anti-injection, confirmations humaines.

7. **Outils externes**
   Google Sheets, CRM, API de livraison, email, base de donnees, calendrier.

8. **Sortie**
   Message, mise a jour, rapport, action metier, notification.

## 13. Securite et risques

Un agent autonome est puissant. Donc il doit etre controle.

Les risques principaux :

- Prompt injection.
- Mauvaises instructions.
- Suppression accidentelle de donnees.
- Mauvaise interpretation d'une demande.
- Acces non autorise.
- Numero WhatsApp bloque ou limite.
- API indisponible.
- Reponse incorrecte au client.

### Comment aegiLabs reduit ces risques

#### 1. RBAC : Role-Based Access Control

Chaque agent a des permissions limitees.

Exemple :

- Un client normal peut demander une information.
- Un manager peut valider un remboursement.
- Seul un numero autorise peut donner des ordres sensibles.

#### 2. Append-only database

L'agent peut ajouter des informations, mais ne peut pas supprimer les donnees.

Exemple :

- Il peut ajouter une nouvelle commande.
- Il ne peut pas effacer toute la base.

#### 3. Human-in-the-loop

Pour les actions sensibles, un humain doit confirmer.

Exemples :

- Remboursement.
- Suppression.
- Changement de prix.
- Action financiere.
- Envoi massif de messages.

#### 4. Confirmation client

Avant de valider une commande, l'agent doit resumer :

> Produit : X  
> Adresse : Y  
> Telephone : Z  
> Prix : W  
> Est-ce correct ?

Cela reduit les erreurs.

#### 5. Logs et monitoring

Chaque action doit etre tracee :

- Qui a demande ?
- Qu'a compris l'agent ?
- Quelle action a ete executee ?
- Quand ?
- Avec quel resultat ?

#### 6. Delais humains

Si l'agent repond trop vite sur certains canaux, cela peut paraitre artificiel ou risquer des limitations. Il faut prevoir des delais naturels selon le canal utilise.

## 21. Questions difficiles et reponses

### Question : Pourquoi une entreprise ne ferait pas cela elle-meme avec ChatGPT ?

Reponse :

> ChatGPT est un outil generaliste. aegiLabs est un service gere qui connecte l'IA aux vrais outils de l'entreprise, avec des permissions, des workflows, de la maintenance et de la securite. Le client ne veut pas apprendre a prompt. Il veut que le travail soit fait.

### Question : Est-ce juste un chatbot ?

Reponse :

> Non. Un chatbot repond. Un agent aegiLabs execute. Il peut utiliser des APIs, mettre a jour des bases de donnees, qualifier des prospects, creer des tickets et suivre un processus complet.

### Question : Que se passe-t-il si l'IA fait une erreur ?

Reponse :

> Les actions sensibles sont limitees par des permissions. Les bases de donnees critiques peuvent etre en mode append-only. Les operations importantes peuvent demander une validation humaine. L'agent doit confirmer les informations avant validation.

### Question : Le client doit-il apprendre un nouvel outil ?

Reponse :

> Non. C'est justement notre force. L'agent fonctionne dans les outils existants du client. Le service est zero-config pour lui.

### Question : Quel est le modele economique ?

Reponse :

> Frais d'integration pour construire et connecter l'agent, puis abonnement mensuel pour l'hebergement, la maintenance, le monitoring et les ameliorations.

### Question : Pourquoi votre equipe peut reussir ?

Reponse :

> Le projet demande des competences techniques que peu de personnes non techniques possedent : serveurs, APIs, orchestration d'agents, securite, integration et maintenance. Notre avantage est notre capacite a transformer cette complexite en service simple pour le client.

### Question : Est-ce que l'agent peut vraiment tout faire ?

Reponse :

> Il peut executer beaucoup de workflows digitaux, mais toujours dans un cadre defini. Nous ne vendons pas une magie illimitee sans controle. Nous vendons des agents specialises, configures pour des roles precis, avec des permissions et des limites.

## 27. Glossaire simple

### Agent IA

Un systeme IA qui peut comprendre une demande, planifier des actions et utiliser des outils.

### Chatbot

Un systeme qui repond a des messages, souvent avec peu d'actions reelles.

### Workflow

Suite d'etapes pour accomplir une tache.

Exemple :

Message client -> verification stock -> confirmation -> mise a jour fichier -> relance.

### API

Moyen pour deux logiciels de communiquer.

### VPS

Serveur prive virtuel utilise pour heberger l'agent.

### RBAC

Controle d'acces base sur les roles. Tout le monde n'a pas les memes permissions.

### Append-only

Mode ou l'agent peut ajouter des donnees, mais ne peut pas supprimer les anciennes.

### MRR

Monthly Recurring Revenue : revenu mensuel recurrent.

### Setup fee

Frais payes une seule fois pour configurer l'agent.

### Retainer

Abonnement mensuel pour maintenance, monitoring et hebergement.
